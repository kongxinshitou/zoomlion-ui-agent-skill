# Product Page Prompt

Use `zoomlion-ui` to create or refactor a Zoomlion-inspired product center or product detail page.

## Goal

Make the product UI feel like a high-end engineering machinery / industrial equipment website: clear, reliable, technical, global, and conversion-friendly.

## Constraints

- Preserve API calls, route params, product IDs, filters, pagination, query keys, and component props.
- Do not invent product model data, parameters, certifications, or performance claims.
- Keep real product data when available; label placeholder data clearly.
- Use authorized machinery/product imagery only. Otherwise use neutral placeholders.

## Product center structure

1. Header and breadcrumb
2. Compact page hero
3. Category tabs or sidebar filters
4. Product grid
5. Featured solution/product banner
6. Service / parts / consultation CTA
7. Footer

## Product detail structure

1. Breadcrumb
2. Product hero: image, model/category name, summary, CTA
3. Key parameters
4. Features and advantages
5. Application scenarios
6. Gallery/video/downloads
7. Related products
8. Service support CTA

## Visual rules

- Primary CTA: Zoomlion-inspired green.
- Product cards: restrained border, 8–12px radius, subtle hover lift, image scale no more than 1.05.
- Typography: strong title hierarchy, readable technical specs, clean Chinese/English font stack.
- Layout: 1200–1440px desktop container; 3–4 cards desktop, 2 tablet, 1 mobile.

## Required output

Report changed files, preserved logic, assumptions, and checklist results.
